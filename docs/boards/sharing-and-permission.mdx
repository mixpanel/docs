## Overview

Reports are always saved to boards. When you save a report, you'll be prompted to save it to a specific board. In essence, the board now acts as a folder, with many potential purposes, including:

- A centralized place to track numerous metrics related to a product launch
- A chain of analyses digging into a particular behavior or issue
- A set of draft reports you're working on

This will make it easy to keep reports organized, and reduce time wasted recreating the same reports again and again.

## Sharing

Every Boards is accessible by all project members. Board-level sharing settings control how Boards are surfaced in Mixpanel and who can edit.

Sharing is done at the board level. All reports saved on a board inherit the same sharing settings of that board. To share a board, click the Share button at the top right of the board.

You can set access for specific individuals, Teams, and everyone in the project.

**Individuals and Teams Sharing:**

- Viewers: Board discoverable in Mixpanel search. Any changes Viewers make will not be saved
- Editors: Able to make and save changes to the Board

**Project Sharing:**

- Off: The Board does not surface in Mixpanel for users who are not Viewers or Editors of the Board. Note, even when project sharing is Off, any project member with the direct link can still view the Board
- Viewer: Board discoverable in Mixpanel. Any changes Viewers make will not be saved
- Editor: Able to make and save changes to the Board

The **Copy URL** button copies a short link to a Board or report. Anyone with the link who has access to the project can open it. What they can do is always governed by the permissions above.

If a Board contains sensitive data, restrict access to the underlying data through [Data Views and Data Classification](https://docs.mixpanel.com/docs/data-governance/data-views-and-classification).

Default settings for project sharing can be set by admins in Project Settings. 

## Linked Reports

Every report in Mixpanel has a board where it was originally saved to. This is called the original board. However, in the interest of being able to use the same report across a variety of boards and sharing the same insight across your team and company, Mixpanel has introduced the concept of a "linked report". A linked report is denoted by the arrow in the icon of the report. Any time an existing report is added to a board or a copy is made, a linked report is made. Any changes to the report will get reflected in all of the linked reports associated with the report.
